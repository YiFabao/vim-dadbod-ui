# Saved Queries 目录树功能

## 功能概述

本次更新为 saved queries 添加了完整的文件管理功能，支持：

1. **展开/收缩文件夹** - 可以点击或使用 `o` 键展开/收缩文件夹
2. **多层级目录** - 支持嵌套的文件夹结构
3. **添加目录** - 按 `A` 键添加新的文件夹
4. **添加文件** - 按 `a` 键添加新的查询文件
5. **删除目录/文件** - 按 `d` 键删除文件夹和文件（文件夹会递归删除）
6. **实时搜索** - 按 `/` 键搜索文件，支持中文，自动展开匹配目录

## 搜索功能

### 使用方式

1. 按 `/` 进入搜索模式
2. 直接输入搜索关键词（支持中文）
3. 搜索结果**实时更新**，每输入一个字符就过滤一次
4. 匹配的文件会自动展开其父目录
5. 按 **Enter** 确认，光标跳到第一个匹配的文件
6. 按 **Esc** 或 **Ctrl-C** 取消搜索

### 搜索特性

- ✅ **实时过滤** - 打字过程中就更新结果，无需按 Enter
- ✅ **支持中文** - 完全支持 CJK 字符输入
- ✅ **自动展开** - 包含匹配文件的目录会自动展开
- ✅ **只显匹配项** - 搜索时只显示匹配的文件和目录
- ✅ **Backspace** - 支持退格删除搜索字符

## 实现细节

### 1. 数据结构变更

在 `db.saved_queries` 中添加了 `dir_expanded` 字段来跟踪每个目录的展开状态：

```vim
db.saved_queries = {
  'expanded': 0,
  'list': [...],
  'dir_expanded': {
    'connection': 1,
    'ops/report': 0,
    ...
  }
}
```

### 2. 新增函数

- `render_query_tree()` - 递归渲染目录树
- `toggle_saved_query_dir()` - 切换目录展开/收缩状态
- `add_saved_query_directory()` - 添加新目录
- `add_saved_query_file()` - 添加新文件
- `add_saved_query_file_in_dir()` - 在指定目录添加文件
- `add_saved_query_subdirectory()` - 在指定目录添加子目录
- `delete_saved_query_item()` - 删除目录或文件
- `build_saved_query_dir_path()` - 构建目录完整路径
- `delete_recursive()` - 递归删除目录
- `search_files()` - 实时搜索文件
- `clear_search()` - 清除搜索
- `tree_has_matches()` - 检查目录树是否包含匹配项

### 3. 文件结构

Saved queries 现在支持多层级目录：

```
~/.local/share/db_ui/
└── mydb/
    ├── connection/
    │   ├── query1.sql
    │   └── query2.sql
    ├── ops/
    │   ├── report/
    │   │   ├── daily.sql
    │   │   └── weekly.sql
    │   └── maintenance.sql
    └── user/
        └── search.sql
```

## 使用方法

### 展开/收缩目录

- 将光标移到目录上，按 `o` 或点击展开/收缩
- 使用 `<C-n>` 进入子节点，`<C-p>` 返回父节点
- 使用 `J` / `K` 在同级目录间导航

### 添加目录

- 将光标移到要添加子目录的父目录上
- 按 `A` 键
- 输入子目录名称
- 目录会在当前选中目录下创建

### 添加文件

- 将光标移到要添加文件的目标目录上
- 按 `a` 键
- 输入文件名称
- 文件会在当前选中目录下创建

### 删除

- 将光标移到要删除的项目上（文件或目录）
- 按 `d` 键
- 确认删除操作

**注意**: 删除目录会递归删除其中所有文件和子目录！

### 重命名

- 将光标移到要重命名的文件上
- 按 `r` 键
- 输入新名称

## 保存查询

在 SQL 编辑窗口中按 `<Leader>W` 保存查询时：

1. 会弹出目录选择对话框
2. 显示所有可用目录（包括嵌套目录）
3. 输入数字选择目标目录
4. 输入文件名保存

如果没有目录存在，会直接保存到根路径。

## 技术实现

### 目录树构建逻辑

1. `load_saved_queries()` 扫描 `save_path` 下的所有文件和目录
2. `_render_saved_queries_section()` 构建目录树结构
3. `render_query_tree()` 递归渲染每个节点
4. 搜索时会调用 `tree_has_matches()` 预检查匹配

### 展开状态管理

目录的展开状态存储在 `db.saved_queries.dir_expanded` 字典中，使用完整的相对路径作为键。

### 搜索实现

- 使用 `getcharstr()` 获取按键（支持多字节/中文字符）
- 实时渲染过滤结果
- 自动展开包含匹配文件的目录

### 删除逻辑

- 对于文件：直接调用 `delete()`
- 对于目录：调用 `delete_recursive()` 递归删除所有内容

## 注意事项

1. 删除文件夹会递归删除其中所有文件
2. 删除操作会弹出确认对话框
3. 目录展开状态会在刷新后保持
4. 只支持在 `g:db_ui_save_location` 配置的目录下操作
5. 搜索支持中文和 Unicode 字符
6. 搜索时只显示匹配的文件和目录
